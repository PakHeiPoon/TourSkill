# TourSkill：构建去中心化的智能体对智能体（A2A）旅游商业新范式

## 执行摘要

TourSkill 是一个受比特币白皮书启发的去中心化旅游商业协议，旨在解决当前在线旅游平台（OTA）对市场的垄断和寻租行为。该项目通过建立一个去中心化的“黄页”注册表，允许商家直接发布可执行的“技能”（Executable Skills），从而实现从“依赖平台”到“智能体自由”的跨越。通过引入模型上下文协议（MCP）和去中心化推理网络，TourSkill 能够让用户的 AI 智能体直接与商家的服务接口进行交互，在无需中介干预的情况下完成查询、预订和支付流程。

---

## 核心主题深度分析

### 1. 现状：OTA 垄断与“选择的幻觉”
当前的旅游市场被中心化平台深度控制。虽然商家表面上拥有定价权，但实际上，平台的发现机制、排名算法和复杂的优惠券生态系统主导了最终成交价。
*   **价格扭曲示例：** 酒店标价 ¥800 的房间，在平台上可能显示为 ¥1,200。平台通过发放“¥200 优惠券”，让消费者误以为以 ¥1,000 的价格获得了实惠，而商家最终可能仅收到 ¥900，差额被平台作为“租金”提取。
*   **控制权：** 平台通过掌控连接点，成为了用户与商家之间必须经过的关卡。

### 2. 愿景：受比特币启发的点对点（P2P）商业
TourSkill 的核心愿景是应用比特币的点对点交易原则——**无需可信第三方的直接交互**。
*   **去中心化注册表：** 作为一个开放、可验证的去中心化索引，商家在此发布其服务，而非依赖中心化平台的审核与分配。
*   **智能体对话（A2A）：** 愿景的核心在于让“你的智能体与他们的智能体直接对话”。这种模式将互联网从单纯的“信息连接”提升为“服务直接调度”。

### 3. 技术范式转换：从静态列表到可执行技能
不同于传统平台的静态信息展示，TourSkill 要求商家发布的是**可执行技能**。这意味着 AI 智能体不仅能读取信息，还能直接调用功能完成业务逻辑。

## 关键引言与背景

> **“我们发明了互联网来直接连接人与人。然后我们又建了平台，坐在每个连接中间收租。”**
*   **语境：** 这句话概括了 TourSkill 挑战现状的根本动机。它指出技术演进的悖论：互联网本应去中心化，但目前的商业模式却在每个连接点制造了新的“租金提取者”。

> **“无需可信第三方的点对点交易。”**
*   **语境：** 引用自比特币白皮书的核心思想。TourSkill 将这一金融领域的原则扩展到了旅游商业领域，通过技术手段消除对中心化媒介的需求。

## 系统架构与技术流程

### 1. 商家技能系统（Merchant Skill System）
商家通过 TourSkill 注册表发布具体的 API 接口，供 AI 智能体调用。

| 类别 | 核心技能 (Skills) | 功能说明 |
| :--- | :--- | :--- |
| **餐厅** | `get_menu`, `reserve_table`, `check_table_availability`, `get_dietary_options` | 提供实时价格、饮食标签及过敏原信息 |
| **酒店** | `check_availability`, `get_rates`, `create_booking`, `get_cancellation_policy` | 处理房型查询、动态定价及取消规则 |
| **景点** | `check_ticket_inventory`, `get_opening_hours`, `purchase_ticket`, `get_visitor_guide` | 票务库存管理、开放时间及交通指南 |

### 2. 技术栈架构

| 层级 | 采用技术 |
| :--- | :--- |
| **前端** | React 19, TypeScript, Vite, Tailwind CSS |
| **智能合约** | Solidity 0.8.24, Hardhat 3 |
| **后端/网关** | FastAPI, Supabase, **MCP (模型上下文协议)** |
| **AI 推理** | 0G Compute Network, @0glabs/0g-serving-broker |
| **区块链** | 0G Network (Testnet & Mainnet), MetaMask, ethers.js v6 |

### 3. 用户流程与智能体交互逻辑
1.  **身份与资金：** 用户连接钱包（如 MetaMask），通过钱包支付 LLM 推理费用。
2.  **发现：** 智能体访问去中心化注册表，验证商家的 Profile Hash。
3.  **决策：** 自主智能体根据用户需求，在最多 8 轮迭代中决定调用哪些工具（技能）。
4.  **执行：** 实时展示工具调用日志，通过 MCP 协议与商家接口通信。
5.  **支付：** 自动监测余额并在必要时进行智能充值。

## 核心特性汇总

*   **去中心化注册表：** 链上身份认证，确保商家信息的真实性与不可篡改性。
*   **用户驱动型 AI：** 无需中心化 API Key，推理费用由用户钱包直接结算。
*   **多城市数据覆盖：** 已接入杭州、上海、苏州、北京的 29 家真实商家数据。
*   **自主性：** LLM 能够独立选择并组合 12 种不同的商家技能来完成复杂任务。

### 行动洞察
1.  **协议标准化：** 采用 MCP 协议是关键，这确保了任何遵循该标准的 AI 智能体都能无缝接入旅游生态。
2.  **成本透明化：** 通过去中心化推理网络，用户直接为计算付费，消除了传统平台隐藏的佣金结构。
3.  **直接商业化：** 商家不再需要为了排名支付竞价排名费用，而是通过提供高质量、可直接调用的“技能”来吸引流量。
